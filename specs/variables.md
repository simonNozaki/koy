# 機能仕様: 変数宣言と代入

**作成日**: 2026-03-26
**ステータス**: 安定

## ユーザーシナリオとテスト

### ユーザーストーリー 1 - イミュータブル変数宣言（優先度: P1）

ユーザーが `val` で変数を宣言し、後からその値を参照する。

**この優先度の理由**: 言語の基本プリミティブ。他のすべての機能が変数束縛に依存する。

**独立テスト**: `val x = 10;` を評価して `interpreter.getValue("x")` が `Value.Int(10)` を返すことを確認する。

**アクセプタンスシナリオ**:

1. **Given** `val x = 42;`、**When** 評価する、**Then** `x` が `Value.Int(42)` に束縛される
2. **Given** `val s = "hello";`、**When** 評価する、**Then** `s` が `Value.String("hello")` に束縛される
3. **Given** `val x = 1;` の後に `x = 2;`、**When** 評価する、**Then** ランタイム例外が発生する

---

### ユーザーストーリー 2 - ミュータブル変数宣言（優先度: P1）

ユーザーが `mutable val` で再代入可能な変数を宣言し、値を更新する。

**この優先度の理由**: ループや状態を持つ計算に必須。

**独立テスト**: `mutable val i = 0;` の後に `i = 1;` を評価して `i` が `1` であることを確認する。

**アクセプタンスシナリオ**:

1. **Given** `mutable val i = 0;` の後に `i = 10;`、**When** 評価する、**Then** `i` が `Value.Int(10)` になる
2. **Given** `mutable val i = 0;` を `while` ループ内でインクリメントする、**When** ループ終了後、**Then** 最終値がすべてのインクリメントを反映する

---

### ユーザーストーリー 3 - 識別子の参照（優先度: P1）

ユーザーが式の中で以前に宣言した変数名を参照する。

**この優先度の理由**: 識別子の解決がなければ、宣言後に変数を使用できない。

**独立テスト**: `val x = 5; val y = x + 1;` を評価して `y` が `6` であることを確認する。

**アクセプタンスシナリオ**:

1. **Given** `val x = 5;` の後に `val y = x + 1;`、**When** 評価する、**Then** `y` が `Value.Int(6)` になる
2. **Given** 未宣言の識別子を参照する、**When** 評価する、**Then** ランタイム例外が発生する

---

### エッジケース

- イミュータブル `val` への再代入は `KoyLangRuntimeException` を発生させなければならない。
- ネストされたスコープでの変数のシャドーイングは現時点では未定義。
- ある式で宣言された変数は、同一インタープリタインスタンス内の後続の式から参照できる。

## 要件

### 機能要件

- **FR-001**: `val name = expr;` は `expr` の評価値を `name` にイミュータブルに束縛しなければならない
- **FR-002**: `mutable val name = expr;` は `name` を再代入可能として束縛しなければならない
- **FR-003**: `name = expr;` は既存の `mutable val` を再代入しなければならない。`val` に対しては例外を発生させる
- **FR-004**: 識別子の参照は環境内で現在束縛されている値に解決されなければならない

## 成功基準

### 測定可能な成果

- **SC-001**: `val` の宣言と参照が成功する
- **SC-002**: `val` への再代入が例外を発生させる
- **SC-003**: `mutable val` を複数回再代入できる

## 前提条件

- 変数スコープは単一のインタープリタセッション内でフラット（ブロックスコープなし）
- トップレベルの `val` 定義は `TopLevel.ValDefinition` としても記述できる
