# 機能仕様: 制御フロー

**作成日**: 2026-03-26
**ステータス**: 安定

## ユーザーシナリオとテスト

### ユーザーストーリー 1 - if 式（優先度: P1）

ユーザーが `if` 式を記述し、真偽値の条件に基づいて一方のブランチを評価する。

**この優先度の理由**: あらゆる条件分岐に必須。

**独立テスト**: `if (true) { 1 } else { 2 }` を評価して結果が `1` であることを確認する。

**アクセプタンスシナリオ**:

1. **Given** `x = 3` で `if (x < 5) { "small" } else { "large" }`、**When** 評価する、**Then** `Value.String("small")` を返す
2. **Given** `x = 10` で `if (x < 5) { "small" } else { "large" }`、**When** 評価する、**Then** `Value.String("large")` を返す
3. **Given** else なしの `if (false) { 1 }`、**When** 評価する、**Then** `Value.Nil` を返す

---

### ユーザーストーリー 2 - while 式（優先度: P1）

ユーザーが `while` ループを記述し、条件が真である間ボディを繰り返し実行する。

**この優先度の理由**: 基本的な繰り返しのプリミティブ。

**独立テスト**: `mutable val i = 0; while (i < 10) { i = i + 1; }` を評価して `i` が `10` になることを確認する。

**アクセプタンスシナリオ**:

1. **Given** `mutable val i = 0; while (i < 10) { i = i + 1; }`、**When** 評価する、**Then** `i` が `Value.Int(10)` になる
2. **Given** 最初から条件が偽のループ、**When** 評価する、**Then** ボディは一度も実行されず `Value.Nil` を返す

---

### ユーザーストーリー 3 - for-in 式（優先度: P2）

ユーザーが `for (i in from to to) { body }` で整数範囲をイテレートする。

**この優先度の理由**: 手動 `while` より直感的な範囲イテレーション。`while` 式に脱糖される。

**独立テスト**: `for (i in 0 to 3) { println(i); }` で `0`、`1`、`2` が出力されることを確認する。

**アクセプタンスシナリオ**:

1. **Given** `for (i in 0 to 5) { println(i); }`、**When** 評価する、**Then** `0, 1, 2, 3, 4` が出力される（上限は含まない）
2. **Given** `for (i in 0 to 0) { body }`、**When** 評価する、**Then** ボディは実行されない
3. **Given** for-in ボディ内でカウンタ変数 `i` を参照する、**When** 評価する、**Then** 現在のイテレーション値が得られる

---

### エッジケース

- `else` なしの `if` は条件が偽のとき `Value.Nil` を返す。
- `for-in` のカウンタは各イテレーションで自動的に 1 増加する。ボディ内で手動インクリメントすると二重インクリメントになる。
- 条件が永遠に真のままの `while` は無限ループになる（タイムアウト保護なし）。
- すべての制御フロー構文は式であり、実行されたブロックの最後に評価された値を返す。

## 要件

### 機能要件

- **FR-001**: `if (cond) { then } else { else }` は対応するブランチを評価して返さなければならない
- **FR-002**: `else` なしの `if (cond) { then }` は条件が偽のとき `Value.Nil` を返さなければならない
- **FR-003**: `while (cond) { body }` は条件が `Value.Bool(true)` の間ボディを繰り返し実行しなければならない
- **FR-004**: `for (i in from to to) { body }` は `i` を `from`（含む）から `to`（含まない）まで 1 ずつ増加しながらイテレートしなければならない
- **FR-005**: すべての制御フロー構文は実行されたブロックの最後に評価された値を返さなければならない

## 成功基準

### 測定可能な成果

- **SC-001**: `if/else` が正しいブランチを選択して値を返す
- **SC-002**: `while` が条件が偽になったとき正しく終了する
- **SC-003**: `for-in` が手動カウンタ管理なしに期待通りの回数イテレートする

## 前提条件

- ループボディは単一の `line` 式（`{ ... }` ブロックも可）
- `for-in` はステップ 1 の整数範囲のみサポート
- `break` や `continue` 構文は存在しない
