# 機能仕様: 演算子

**作成日**: 2026-03-26
**ステータス**: 安定

## ユーザーシナリオとテスト

### ユーザーストーリー 1 - 算術演算子（優先度: P1）

ユーザーが整数値に対して `+`、`-`、`*`、`/`、`%` を使った算術式を記述する。

**この優先度の理由**: 最も基本的な計算。ほぼすべてのプログラムが必要とする。

**独立テスト**: `3 * 5 + 10` を評価して結果が `25` であることを確認する。

**アクセプタンスシナリオ**:

1. **Given** `10 + 20`、**When** 評価する、**Then** `Value.Int(30)` を返す
2. **Given** `3 * 5 + 10`、**When** 評価する、**Then** `Value.Int(25)` を返す（標準的な演算子優先度）
3. **Given** `10 / 2`、**When** 評価する、**Then** `Value.Int(5)` を返す
4. **Given** `10 % 3`、**When** 評価する、**Then** `Value.Int(1)` を返す
5. **Given** `"Hello, " + "Koy"`、**When** 評価する、**Then** `Value.String("Hello, Koy")` を返す

---

### ユーザーストーリー 2 - 比較演算子（優先度: P1）

ユーザーが `==`、`!=`、`<`、`<=`、`>`、`=>` を使って二つの値を比較する。

**この優先度の理由**: 条件式やループ条件に必須。

**独立テスト**: `3 < 5` を評価して `Value.Bool(true)` であることを確認する。

**アクセプタンスシナリオ**:

1. **Given** `3 < 5`、**When** 評価する、**Then** `Value.Bool(true)` を返す
2. **Given** `5 == 5`、**When** 評価する、**Then** `Value.Bool(true)` を返す
3. **Given** `5 != 3`、**When** 評価する、**Then** `Value.Bool(true)` を返す
4. **Given** `5 <= 5`、**When** 評価する、**Then** `Value.Bool(true)` を返す

---

### ユーザーストーリー 3 - 論理演算子（優先度: P1）

ユーザーが `and` / `or` を使って真偽値式を結合する。

**この優先度の理由**: 複合条件に必須。

**独立テスト**: `true and true` を評価して `Value.Bool(true)` であることを確認する。

**アクセプタンスシナリオ**:

1. **Given** `true and true`、**When** 評価する、**Then** `Value.Bool(true)` を返す
2. **Given** `true and false`、**When** 評価する、**Then** `Value.Bool(false)` を返す
3. **Given** `true or false`、**When** 評価する、**Then** `Value.Bool(true)` を返す
4. **Given** `false or false`、**When** 評価する、**Then** `Value.Bool(false)` を返す

---

### ユーザーストーリー 4 - 単項インクリメント / デクリメント（優先度: P2）

ユーザーが `++name` または `--name` をミュータブル変数に適用して値を増減させる。

**この優先度の理由**: カウンタ更新の便利な短縮記法。

**独立テスト**: `mutable val n = 1; val n2 = ++n;` を評価して `n2` が `2` であることを確認する。

**アクセプタンスシナリオ**:

1. **Given** `mutable val n = 1; val r = ++n;`、**When** 評価する、**Then** `r` が `Value.Int(2)` で `n` も `Value.Int(2)`
2. **Given** `mutable val m = 1; val r = --m;`、**When** 評価する、**Then** `r` が `Value.Int(0)` で `m` も `Value.Int(0)`

---

### ユーザーストーリー 5 - コレクション演算子（優先度: P2）

ユーザーが `collection->index` でインデックスアクセスし、`collection <- element` で要素を追加する。

**この優先度の理由**: 配列・セットの操作を可能にする。

**独立テスト**: `val a = [10, 20, 30]; val x = a->1;` を評価して `x` が `20` であることを確認する。

**アクセプタンスシナリオ**:

1. **Given** `val a = [10, 20, 30];` の後に `a->0`、**When** 評価する、**Then** `Value.Int(10)` を返す
2. **Given** `val a = [1, 2];` の後に `a <- 3`、**When** 評価する、**Then** `a` が 3 要素を含む

---

### エッジケース

- ゼロ除算の動作は現時点では未定義。
- `%` は剰余演算子とセットリテラルのプレフィックスを兼用する。文脈によってパースが決まる。
- 演算子優先度: `*`、`/`、`%` は `+`、`-` より強く結合し、比較演算子は最も弱く結合する。

## 要件

### 機能要件

- **FR-001**: 算術演算子は `Value.Int` を受け取り `Value.Int` を返さなければならない
- **FR-002**: `+` を二つの `Value.String` に適用した場合は連結しなければならない
- **FR-003**: 比較演算子は `Value.Bool` を返さなければならない
- **FR-004**: 論理演算子 `and` / `or` は `Value.Bool` を受け取り `Value.Bool` を返さなければならない
- **FR-005**: `++` と `--` は `mutable val` の識別子にのみ適用できる
- **FR-006**: `->` はゼロ始まりのインデックスでコレクション要素にアクセスしなければならない
- **FR-007**: `<-` はコレクションに要素を追加しなければならない

## 成功基準

### 測定可能な成果

- **SC-001**: すべての算術演算子が正しい整数結果を返す
- **SC-002**: すべての比較演算子が正しい真偽値を返す
- **SC-003**: 論理演算子が正しく評価される

## 前提条件

- 演算子優先度は標準的な数学的慣例に従う（乗除算 > 加減算 > 比較 > 論理）
- Int と String の間に暗黙的な型変換はない
