# 機能仕様: 関数

**作成日**: 2026-03-26
**ステータス**: 安定

## ユーザーシナリオとテスト

### ユーザーストーリー 1 - トップレベル関数の定義と呼び出し（優先度: P1）

ユーザーが `fn` でトップレベルに名前付き関数を定義し、`main` から呼び出す。

**この優先度の理由**: Koy プログラムを構造化する主要な手段。`main` がプログラムのエントリーポイント。

**独立テスト**: `fn add(v1, v2) { v1 + v2; }` と `fn main() { add(10, 20); }` を定義して `callMain` が `Value.Int(30)` を返すことを確認する。

**アクセプタンスシナリオ**:

1. **Given** `fn add(v1, v2) { v1 + v2; }` と `fn main() { add(10, 20); }`、**When** `callMain` を呼ぶ、**Then** `Value.Int(30)` を返す
2. **Given** 再帰関数 `fn factorial(v) { if (v < 2) { 1 } factorial(v-1) * v; }`、**When** `5` で呼ぶ、**Then** `Value.Int(120)` を返す
3. **Given** 引数の数が合わない関数呼び出し、**When** 評価する、**Then** 未束縛のパラメータは `Value.Nil` に解決される

---

### ユーザーストーリー 2 - 関数リテラル（ラムダ）（優先度: P1）

ユーザーが `|args| { body }` で無名関数を定義し、変数に代入する。

**この優先度の理由**: 第一級関数と高階プログラミングパターンを実現する。

**独立テスト**: `val square = |n| { n * n; };` を定義して `square(5)` が `Value.Int(25)` を返すことを確認する。

**アクセプタンスシナリオ**:

1. **Given** `val square = |n| { n * n; };`、**When** `square(5)` を呼ぶ、**Then** `Value.Int(25)` を返す
2. **Given** `val add = |x, y| { x + y; };`、**When** `add(3, 4)` を呼ぶ、**Then** `Value.Int(7)` を返す
3. **Given** `val` で束縛された関数リテラルをオブジェクトプロパティとして格納する、**When** メソッド呼び出し構文でアクセスする、**Then** 呼び出せる

---

### ユーザーストーリー 3 - ラベル付き引数呼び出し（優先度: P2）

ユーザーが `fn[param = value]` の構文でラベル付き引数を使って関数を呼び出す。

**この優先度の理由**: 複数パラメータがある呼び出しの可読性を向上させる。

**独立テスト**: `factorial[x = 5]` が `factorial(5)` と同じ結果を返すことを確認する。

**アクセプタンスシナリオ**:

1. **Given** `fn factorial(x) { ... }`、**When** `factorial[x = 5]` で呼ぶ、**Then** `factorial(5)` と同じ結果を返す

---

### ユーザーストーリー 4 - メソッド呼び出し（優先度: P2）

ユーザーがオブジェクトのプロパティとして格納された関数をドット記法 `obj.method(args)` で呼び出す。

**この優先度の理由**: 関数プロパティを持つオブジェクトによるオブジェクト指向パターンを実現する。

**独立テスト**: `val obj = { greet: |msg| { "Hello, " + msg; } };` を定義して `obj.greet("Koy")` が `Value.String("Hello, Koy")` を返すことを確認する。

**アクセプタンスシナリオ**:

1. **Given** `val obj = { print: |msg| { "Hello, " + msg; } };`、**When** `obj.print("Koy")` を呼ぶ、**Then** `Value.String("Hello, Koy")` を返す
2. **Given** 関数でないプロパティをメソッド呼び出し構文でアクセスする、**When** 評価する、**Then** ランタイム例外が発生する

---

### エッジケース

- `main` 関数を持たないプログラムは `callMain` で実行できない。
- 再帰呼び出しはサポートされる。末尾呼び出し最適化は未定義。
- 関数リテラルが外側のミュータブル変数をクロージャとしてキャプチャするかどうかは未定義。
- パラメータなしの関数は空の引数リストを使う: `fn noArgs() { ... }` / `|| { body }`。

## 要件

### 機能要件

- **FR-001**: `fn name(args) { body }` はトップレベルに名前付き関数を定義しなければならない
- **FR-002**: `callMain` は引数なしで `main` という名前の関数を呼び出さなければならない
- **FR-003**: `|args| { body }` は格納・呼び出し可能な関数値を生成しなければならない
- **FR-004**: `name(args)` は名前で関数を検索し、評価済みの引数で呼び出さなければならない
- **FR-005**: `name[label = expr, ...]` はラベルで引数を照合してラベル付き呼び出しを実行しなければならない
- **FR-006**: `obj.method(args)` は `obj` の関数プロパティとして `method` を解決して呼び出さなければならない

## 成功基準

### 測定可能な成果

- **SC-001**: トップレベル関数が互いに呼び出せる（相互再帰・前方参照）
- **SC-002**: 関数リテラルを `val` に格納して呼び出せる
- **SC-003**: 再帰関数が正しい結果を返す（例: `factorial(5) = 120`）
- **SC-004**: 関数プロパティを持つオブジェクトへのメソッド呼び出しが正しい値を返す

## 前提条件

- `main` 関数は引数を取らない
- 関数定義のパラメータ名は単純な識別子
- ラベル付き呼び出しの引数の順序は関数定義のパラメータの順序と一致しなければならない
